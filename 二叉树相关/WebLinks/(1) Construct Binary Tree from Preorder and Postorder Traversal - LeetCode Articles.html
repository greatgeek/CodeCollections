889. Construct Binary Tree from Preorder and Postorder Traversal 

Average Rating: 3.40 (40 votes)

Aug. 18, 2018  |  24.8K views

Return any binary tree that matches the given preorder and postorder traversals.

Values in the traversals pre and post are distinct positive integers.

 

Example 1:

Input: pre = [1,2,4,5,3,6,7], post = [4,5,2,6,7,3,1]
Output: [1,2,3,4,5,6,7]

 

Note:

  • 1 <= pre.length == post.length <= 30
  • pre[] and post[] are both permutations of 1, 2, ..., pre.length.
  • It is guaranteed an answer exists. If there exists multiple answers, you can return any of them.

Solution

Approach 1: Recursion

Intuition

A preorder traversal is:

  • (root node) (preorder of left branch) (preorder of right branch)

While a postorder traversal is:

  • (postorder of left branch) (postorder of right branch) (root node)

For example, if the final binary tree is [1, 2, 3, 4, 5, 6, 7] (serialized), then the preorder traversal is [1] + [2, 4, 5] + [3, 6, 7], while the postorder traversal is [4, 5, 2] + [6, 7, 3] + [1].

If we knew how many nodes the left branch had, we could partition these arrays as such, and use recursion to generate each branch of the tree.

Algorithm

Let's say the left branch has LL nodes. We know the head node of that left branch is pre[1], but it also occurs last in the postorder representation of the left branch. So pre[1] = post[L-1] (because of uniqueness of the node values.) Hence, L = post.indexOf(pre[1]) + 1.

Now in our recursion step, the left branch is represnted by pre[1 : L+1] and post[0 : L], while the right branch is represented by pre[L+1 : N] and post[L : N-1].

Approach 2: Recursion (Space Saving Variant)

Explanation

We present a variation of Approach 1 that uses indexes to refer to the subarrays of pre and post, instead of passing copies of those subarrays. Here, (i0, i1, N) refers to pre[i0:i0+N], post[i1:i1+N].

Both approaches can be improved with caching, here's Approach 2:

class Solution(object):
    def constructFromPrePost(self, pre, post):
        def make(i0, i1, N):
            if N == 0: return None
            root = TreeNode(pre[i0])
            if N == 1: return root

            L = idx[pre[i0+1]] - i1 + 1
            
            root.left = make(i0 + 1, i1, L)
            root.right = make(i0 + L + 1, i1 + L, N - 1 - L)
            return root
        
        idx = {v: i for i, v in enumerate(post)}
        return make(0, 0, len(pre))

class Solution {

TreeNode* constructFromPrePost(vector& pre, vector& post) {
TreeNode *root = new TreeNode(pre[0]);
TreeNode cur = root;
stack<TreeNode>stk;
stk.push(cur);
int i=1, j=0;

    while(i < pre.size() && j < post.size()) {
        TreeNode* cur = stk.top();
        
        
        
        if(post[j] == cur->val) {
            j++;
            stk.pop();
        }
        
        else if(pre[i] == post[j]) {
            TreeNode* temp = new TreeNode(pre[i]);
            if(!cur->left) cur->left = temp;
            else cur->right = temp;
            i++; j++;
        }
        
        else {
            TreeNode* temp = new TreeNode(pre[i]);
            if(!cur->left) cur->left = temp;
            else cur->right = temp;
            i++;
            stk.push(temp);
        }
    }
    
    return root;
}

If: 
    pre = R (A...) (B...)
    R must be the root, 
    it's not clear whether A is the left child of R or the right child of R.
Case 1: So, if we also have: 
    post = (.......A) R
    then it's clear that:
        option 1: A is the right child of R and R doesn't have left child.
            why? If R also had a left child, it would appear before A (adjacent to R) in pre.
        option 2: A is the left child of R and R doesn't have right child.
            why? If R also had a right child, it would appear after A (before/adjacent to R) in post.
    Let's use option 2.

Case 2: What if instead we have:
    post = (...A)(...B) R
    Then there is only one option:
        A must be the left child of R, B must be the right child of R.
        
So in any case, A is the left child of R.

class Solution:
    def constructFromPrePost(self, pre, post):        
        def recur(low_pre, high_pre, low_post, high_post):
            root_val = pre[low_pre]
            x = TreeNode(root_val)
            if low_pre == high_pre: return x
            left_child = pre[low_pre + 1]
            if left_child == post[high_post-1]: # Case 1: left_child is the only child of x
                x.left = recur(low_pre+1, high_pre, low_post, high_post-1)
            else: # Case 2: x has two children
                right_child = post[high_post-1]
                i_pre = high_pre # find the index of the right child of x in pre:   x (left_child.......)(right_child......)
                while pre[i_pre] != right_child: i_pre -= 1
                j_post = low_post # find the index of the left child of x in post:   (.......left_child)(......right_child) x
                while post[j_post] != left_child: j_post += 1
                x.left = recur(low_pre+1, i_pre-1, low_post, j_post)
                x.right = recur(i_pre, high_pre, j_post+1, high_post-1)
            
            return x
        return recur(0, len(pre)-1, 0, len(post)-1)
